# ADR-000: Record Architecture Decisions

> **状态**: Accepted
> **日期**: 2026-07-14
> **决策者**: Harness 架构设计

---

## Context

本项目是一个长期维护的本地 Agent 编排运行时。架构决策需要在项目过程中被持续追踪、审查和理解。

## Decision

使用 Architecture Decision Records (ADR) 记录所有重大架构决策。

每个 ADR 包含：
- 标题和编号
- 状态（Proposed / Accepted / Deprecated / Superseded）
- 上下文（为什么需要决策）
- 决策内容
- 后果（正面和负面的影响）

## Consequences

- **正面**：决策可追溯，新团队成员可以理解历史背景
- **正面**：在后续阶段审查时，可以判断某个决策是否仍然成立
- **负面**：需要维护文档，需要决策纪律
