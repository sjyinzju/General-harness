## I4.6 Architecture

### Database (migration 023)

```
candidate_snapshots     ← immutable frozen candidate
review_requests         ← lifecycle tracking (CAS)
review_findings         ← structured per-finding records
review_decisions        ← terminal decision records
review_dossier_refs     ← dossier artifact references
review_cache            ← deduplication (composite key)
review_invocation_log   ← invocation counter (real vs cache-hit)
review_events           ← append-only durable events
```

### Review State Machine

```
Requested → Preparing → Prechecking → Reviewing → Approved
                                           ↓          ↓
                                        Blocked    Rejected
                             ↓            ↓           ↓
                        Cancelled     (terminal)  (terminal)
                             ↓
                          Stale
```

### Decision Policy (I4.6 strict)

```
0 Findings + Approved decision → Approved
Any Finding                     → Rejected
Unparseable output              → Blocked
Precheck failure                → Blocked
Reviewer timeout/crash          → Blocked
Candidate changed               → Stale
User cancellation               → Cancelled
```

## Known Boundaries

1. **Real Reviewer Smoke**: Requires live Claude/Codex binary in runtime environment with Discovery/RuntimeProfile. FakeReviewer covers all 7 smoke scenarios deterministically. Real smoke is deferred to environment where Agent binaries are configured.

2. **Read-only Layer 1 (policy enforcement)**: File-system-level write protection is not implemented. Layer 2 (digest-based detection) catches any modification — rogue reviewer modifications are detected and result in Stale/Blocked, never Approved.

3. **I5 Integration Queue**: Contract defined and tested. Full I5 queue implementation is the next milestone.
