#### Domain and State Machine
- Candidate digest determinism
- Candidate diff/task_spec/base_commit changes digest
- ReviewState terminal classification
- All legal transitions (Requested→Preparing→Prechecking→Reviewing→Approved)
- Illegal transitions rejected (skip steps, terminal→non-terminal)
- Cancelled/Stale from any non-terminal
- Re-review requires new ReviewRequest

#### Candidate
- Freeze + retrieve
- Duplicate detection
- Composite digest covers all bound fields

#### Precheck
- All-pass path
- CompletionEligibility failure → Blocked
- Missing workspace → Blocked
- Missing base commit → Blocked

#### Reviewer Selection
- Selects different-from-executor profile
- Returns None when no alternative available
- Prefers structured_output capability

#### Decision Policy
- Zero findings + "Approved" → Approved
- Any finding → Rejected (strict I4.6 policy)
- Critical/High finding → Rejected
- Unparseable decision → Blocked
- Inconsistent (Rejected with no findings) → Blocked

#### Lifecycle
- Full path to Approved
- Full path to Rejected
- Cancel from Requested
- Terminal state immutability
- Stale detection marks active reviews

#### Persistence
- Idempotent candidate creation
- Idempotent review creation
- CAS state transitions
- Finding persistence and retrieval

#### FakeReviewer
- Approved, Rejected, Blocked scripts
- Crash simulation
- Garbage output simulation
