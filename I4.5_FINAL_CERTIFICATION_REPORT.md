## Fixes Applied

### F2 (HIGH): Non-Transactional Lifecycle UPDATE Race

**Root cause**: `ownership_service.rs:328-335` contained a non-transactional
`UPDATE verification_runs SET lifecycle='running'` in the AlreadyOwned path.
This raced with the transactional CAS at line 368-372.  When interleaved
correctly (A wins handoff → B runs AlreadyOwned → B's UPDATE changes
lifecycle/version → A's transactional CAS fails), both sides returned
AlreadyOwned — zero winners.

**Fix**: Removed the non-transactional UPDATE.  The AlreadyOwned path is now
strictly read-only with respect to lifecycle/version.  Only the transactional
CAS may transition the run from 'created' to 'running'.

**Tests added**:
- `test_two_pools_deterministic_interleaving_a_pauses_b_already_owned` —
  forces A to pause after handoff acquisition; B runs AlreadyOwned path;
  verifies B does NOT modify lifecycle; A's CAS succeeds → Acquired.
- `test_two_pools_deterministic_interleaving_b_wins_a_already_owned` —
  reverse ordering; B runs first, wins; A returns AlreadyOwned.

**Stress verification**:
- Deterministic interleaving A→B: 20/20 PASS
- Deterministic interleaving B→A: 20/20 PASS
- Concurrent current_thread: 100/100 PASS
- Concurrent multi_thread: 100/100 PASS

### F1 (HIGH): Suspended Process on Job Assignment Failure

**Root cause**: `manager.rs:150-164` gated resume on `job_object_active()`.
When Job creation/assignment failed (E2-E7 error paths), the process stayed
suspended until timeout cleanup (up to 90s).

**Fix**: Resume unconditionally on Windows (Contract B — Degraded Fallback
per ADR-007).  If Job is active → Job containment.  If Job failed →
taskkill /T /F fallback (already built into ProcessTreeGuard).  Resume
failure → kill suspended process immediately + return error (no orphan).

**Error path coverage**:
- E2-E7 (Job creation/assignment/verification failure): process resumed,
  containment_mode=TaskkillFallback, cancel uses taskkill, no orphan.
- E8 (Resume failure): process killed via Job or taskkill, child.wait()
  confirms death, ProcessSpawnFailed returned, no orphan.
- E9/E10 (Termination failure): taskkill fallback executed, error logged.

**Verification**: Full certification runner — Windows Process Tree 200/200,
Cancellation 5/5, all PASS.

### F4 (LOW): Runner Scenario Evidence

**Root cause**: Runner script hardcoded `-Passed 27` for scenarios_summary.
Scenarios were run as a single test binary invocation; individual results
were not tracked.

**Fix**: Runner now executes all 27 scenarios individually.  Each scenario
produces a result row with test name, binary, passed/failed.  Summary is
derived from actual execution counts — no hardcoded constants.

**Runner fixes (pre-existing)**:
- Emoji characters replaced with ASCII for PowerShell 5.1 compatibility
- `[PSCustomObject]` type for `Measure-Object` compatibility
- `$script:Results` scope fix for function writes
- String→array argument splitting for cargo `--test` flag
- `$ErrorActionPreference = "Continue"` to prevent stderr abort
