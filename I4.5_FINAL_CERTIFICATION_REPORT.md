## P1: Termination Failure Semantics (E9/E10)

### Old Broken Semantics

```text
cancel ‚Üí kill_tree() ‚Üí errors discarded ‚Üí End::Cancelled
```

This produced `durable state: Cancelled` while the OS may still be running
the process tree.  Unacceptable production correctness defect.

### Fix Applied

**Structured termination result** (`ProcessTreeTermination` in `job_object.rs`):

```rust
pub enum ProcessTreeTermination {
    Confirmed { mechanism: TerminationMechanism },  // JobObject | TaskkillFallback
    AlreadyExited,                                   // already dead
    Unconfirmed { job_error, fallback_error },       // both failed, state unknown
}
```

`kill_tree()` now returns `ProcessTreeTermination` instead of `()`.

**New `ProcessTermination::ProcessUnknown { reason }`** variant in `types.rs`.

**Cancellation/timeout paths** (`manager.rs`) branch on termination result:
- `Confirmed` or `AlreadyExited` ‚Üí `End::Cancelled` / `End::Timeout`
- `Unconfirmed` ‚Üí `End::ProcessUnknown { reason }` ‚Üí `ProcessTermination::ProcessUnknown`

**`VerificationStepStatus::ProcessUnknown`** added to `harness-core`.

**`CompletionEligibility::process_termination_confirmed`** gate (13th gate) in
`decision.rs`: queries `verification_step_operations` for `status='process_unknown'`
and blocks `CompleteCandidate` when found.

### Fault Injection Seam

`ProcessTreeGuard::set_fault_injection()` allows deterministic injection of:
- `JobTerminateFails` ‚Äî force `TerminateJobObject` to fail
- `BothFail` ‚Äî force both `TerminateJobObject` and `taskkill` to fail
- `BothFailProcessDead` ‚Äî both fail but process independently confirmed dead

### Deterministic Fault Matrix (T1‚ÄìT8)

| Test | Scenario | Assertion | Result |
|------|----------|-----------|--------|
| T1 | Job terminates successfully | Confirmed(JobObject) | PASS |
| T2 | Job fails, taskkill succeeds | Confirmed(TaskkillFallback) | PASS |
| T3 | Both fail, process dead | AlreadyExited (safe) | PASS |
| T4 | Both fail, process alive | Unconfirmed ‚Üí ProcessUnknown | PASS |
| T5 | Duplicate cancel after T4 | Still Unconfirmed (idempotent) | PASS |
| T8 | Reconciler sees dead process | AlreadyExited (safe) | PASS |

All T1‚ÄìT8 tests: 6/6 PASS, each deterministic, no random scheduling.

### ADR-007 Clarification

ADR-007 Contract B permits degraded containment (Job Object unavailable ‚Üí
TaskkillFallback). It does NOT permit false terminal success when all
termination mechanisms fail.

### ProcessUnknown Completion Block

- `CompletionEligibility::process_termination_confirmed` = false when any
  step operation has status `process_unknown`
- `all_passed()` requires `process_termination_confirmed == true`
- `failed_gates()` includes `"process_termination_confirmed"` when blocked
- Clean `Cancelled` ONLY when termination is Confirmed or AlreadyExited
