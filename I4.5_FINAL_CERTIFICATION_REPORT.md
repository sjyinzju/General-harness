### 4. DeletionGuard — 16 Safety Layers

Every deletion must pass ALL checks:

1. Path exists
2. Not a symlink/junction/reparse-point
3. Canonical path computable (symlink-escape guard)
4. Within managed root (canonical containment)
5. Not the managed root itself
6. Not a protected root (repo, target, user home, system TEMP)
7. Not under shared cargo target (`target/debug`)
8. Not a `.git` directory
9. Ownership marker exists and is valid JSON
10. Schema version matches (v1)
11. Marker `kind` matches managed root
12. `run_id` matches directory name
13. Owner PID not alive (with creation-time comparison)
14. No active execution in registry
15. Active marker beyond grace period (or terminal state beyond TTL)
16. All checks cumulative; any failure = SafetyVerdict::Denied { reasons }

**Never uses a bare `bool` for safety decisions.**

### 5. Current-Run Cleanup

For the Runner's own managed temp directory:

- Directory created atomically with `.harness-owned.json`
- Runner executes with `$env:TEMP` pointing to managed temp
- On completion: marker finalized (`completed`/`failed`)
- Cleanup with bounded retry (100ms, 250ms, 500ms)
- Environment TEMP/TMP restored in `finally` block
- Cleanup failure → directory left for Startup Janitor
- PID reuse guarded by process-creation-time comparison

### 6. Startup Janitor

Runs once at `ProductionGraph` initialization:

- Scans `target/harness-temp/`, `target/harness-evidence/`, `target/harness-cargo-runs/`
- Reclaims stale owned directories beyond grace period
- Preserves: unmarked directories, active directories, within-grace directories
- Evidence retention: keeps latest 3 successful, 1 failed
- Orphan artifacts reclaimed via `ArtifactRoot::reclaim_orphans()`
- Never scans system TEMP or non-managed target subdirectories

### 7. Evidence Retention

- Maximum 3 successful evidence runs kept (per code_head)
- Maximum 1 failed evidence run kept (per code_head)
- Current run evidence always preserved
- Forbidden in evidence: PDB, EXE, RLIB, RMeta, DLL, deps/, incremental/, build/, .fingerprint/
- Only permitted: JSON, JSONL, text logs, small SQLite evidence, failure summaries
- Unmarked historical evidence preserved for manual review
